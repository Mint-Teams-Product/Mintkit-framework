Mintkit LiveServer
Built with
Mint teams

A high-performance, cross-platform live development server written in C and C++ with inline Assembly with native GUI support for Windows, Linux, and macOS.

Liveserver Features

Core Features

  • Real-time file change detection with line-by-line diff
  • High-resolution timing for performance monitoring (microsecond precision)
  • Hot reload with detailed timing statistics
  • Cross-platform GUI with native look

File Watching

  • Tracks HTML, CSS, JS, JSON, TS, TSX, JSX files
  • Shows added (+), removed (-), and modified (~) lines
  • File processing time tracking per file
  • Server response time monitoring

HTTP Endpoints

  • / - Serves static files
  • /reload - Hot reload status endpoint
  • /memory-stats - Memory usage statistics
  • /live-reload.js - Live reload client script

Installation

Prerequisites

Windows

  • MSYS2 or MinGW-w64 with GCC
  • Required libraries: ws2_32, shell32, dwmapi, comctl32

Linux

  • GCC 7+ with C++17 support
  • GTK+ 3.0 development libraries
  • pkg-config

macOS

  • Xcode Command Line Tools or GCC
  • CoreFoundation and ApplicationServices frameworks

Building

Compilation

Windows: 

g++ -std=c++17 -O2 LiveServer.cpp -o LiveServer.exe -lws2_32 -lshell32 -ldwmapi -lcomctl32 -mwindows

Linux: 

# Install dependencies first
sudo apt-get install build-essential libgtk-3-dev pkg-config

# Compile
g++ -std=c++17 -O2 LiveServer.cpp -o LiveServer $(pkg-config --libs gtk+-3.0 gdk-3.0 gio-2.0 gobject-2.0 glib-2.0) $(pkg-config --cflags gtk+-3.0 gdk-3.0 gio-2.0 gobject-2.0 glib-2.0)

macOS: 

g++ -std=c++17 -O2 LiveServer.cpp -o LiveServer -framework CoreFoundation -framework ApplicationServices

Recommend Options

- Debug: -g -DDEBUG -Wall -Wextra - Release: -O3 -DNDEBUG -Wall -Wextra - Normal: -O2 -Wall -Wextra

Performance monitoring

Enhanced Performance
Monitoring for Mintkit

Overview

The MintPuts development server has been enhanced with comprehensive
performance monitoring that replaces CPU cycle counting with high-resolution
millisecond timing. This provides more accurate and meaningful performance metrics
for development workflows.

Key Changes

Server Side Performance Monitoring

WebExecute.c
Before:
  • Used CPU cycles for performance measurement
  • Less accurate across different CPU architectures
  • Harder to interpret for developers
After:
  • Uses QueryPerformanceCounter and
    QueryPerformanceFrequencyfor high-resolution timing
  • All measurements now in milliseconds (ms)
  • More accurate and cross-platform compatible
Enhanced Features:
  • File scan performance tracking
  • Request serving time measurement
  • Memory usage monitoring
  • Real-time performance logging

Client-Side Performance Monitoring

live-reload.js
Enhanced Features:
  • Detailed request timing statistics
  • Average response time calculation
  • Error tracking and reporting
  • Visual indicators with emojis for better readability
  • Performance stats every 10 requests
